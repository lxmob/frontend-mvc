什么是 MVC
MVC 是一种架构设计思想
这种概念最开始是从后端开发引入的，全称是 Model View Controller

M 层做了什么
Model 是数据模型层，是操作数据库，对数据进行增删改查的操作

V 层做了什么
View 是视图层，用来显示视图或视图模板的渲染

C 层做了什么
Controller 是控制层，通过方法逻辑对 M 层与 V 层的操作，核心枢纽

服务端 MVC
通过 V 层发送请求 -> C 层接收请求 -> 让 M 层去操作数据库 -> 交给 C 层 -> 返回到前端

前端 MVC
借鉴的只是在项目设计上的分层思想与方法
这是不同的应用场景，通过 MVC 设计思想可以应用在任何设计中去
后端的 MVC 设计 M 层是对数据库的操作，在进行移植到前端中去会感觉 M 层与 C 层粘黏性很大
而对于前端 MVC 来讲，我们更希望是 M 层是对数据的管理，而请求数据的操作是由 service 来做的
这点对于前端与后端的 MVC 设计思想的区别

前端 Vue - MVC
通过 M 层管理视图数据 -> 数据与视图的关联 -> V 层来渲染视图 -> C 层管理方法逻辑
